# ミキシングの基礎

**ミキシング**は、録音された複数のパートを 1 つのステレオへバランスよくまとめ、意図のある音にする技術です。すべての要素が聞こえ、互いに干渉せず、前後の奥行きと左右の広がりが感じられる状態を目指します。本ページは [ミキシングエンジン](../../mixing.md) ガイドを読む前に必要な言葉を定義します。概念のみで、コードはありません。

::: tip ミキシングの位置
通常は個々のパートを**編集**し（タイミング・ピッチ・ノイズ）、それらを**ミックス**し、仕上がったステレオファイルを配信向けに**マスタリング**します。ミキシングは、別々のステムのフォルダをバランスの取れた 1 曲に変える中間工程です。ここでバランスが取れていれば、マスタリングは救済ではなく仕上げになります。
:::

## トラックとチャンネルストリップ

**トラック**は録音された 1 パートです。リードボーカル、ベースの DI、ステレオのドラムオーバーヘッドなど。ミキサーでは各トラックが**チャンネルストリップ**を通ります。これはそのパートだけを処理する縦 1 列のコントロール群です。ミキシングの技術の本質は、各ストリップをパートがうまく重なるように設定し、共通の出力へルーティングすることにあります。

## レベル: トリム、フェーダー、ゲインステージング

ストリップには 2 つのレベルコントロールがあり、役割が異なります。

- **入力トリム**は、処理の前、入口で妥当な*作業*レベルを設定します。録音は音量がばらばらなので、トリムでそろえ、後段のプロセッサが健全な信号を受け取れるようにします。
- **フェーダー**は、他トラックと比べてバランスを取る*バランス*コントロールです。ボーカルを前に、音楽を後ろに。

各段でレベルを適切に保つこと — 小さすぎず（ノイジー）、大きすぎず（クリップ） — を**ゲインステージング**と呼び、ミキシングで最も重要な習慣です。[ゲインステージング](./gain-staging.md)を参照してください。

## パン: 左右への配置

**パン**はトラックを左いっぱいから右いっぱいまでステレオフィールド内に配置します。パートを広げると、中央に積み重ねる代わりにそれぞれに居場所ができます。2 つの細かい点が重要です。

- **パンモード**は位置をチャンネルへ*どうマッピングするか*を決めます。モノラルソースを動かす本来のパン、ステレオトラックの片側を下げるだけのバランス、左右独立の配置のいずれかです。
- **パンロー**は中央をサイドに対して*どれだけの音量*にするかを決め、パンしてもパートの音量が跳ねないようにします。

## インサートとセンド

初学者が最も取り違える区別ですが、名前がつけば簡単です。

- **インサート**は*直列*に入ります。トラック全体がそこを通ります。そのトラック固有の処理 — ボーカルのコンプ、ベースの EQ — に使います。
- **センド**はトラックの*コピー*を*並列*に共有の行き先へ送ります。複数トラックが共有するエフェクト — ボーカル・スネア・ギターが供給する 1 つのリバーブ — に使い、同じ空間に座らせます。

::: info プリフェーダーとポストフェーダー
センドは信号をフェーダーの**前**（バランスから独立 — ヘッドホン／キューミックス向き）でも**後**（バランスに追従 — フェーダーを動かしてもエフェクトの比率が保たれる）でもタップできます。正しい方を選ぶのは定番のつまずき所で、[ミキシングエンジン](../../mixing.md#チャンネルストリップを信号順にたどる)ページが各タップの位置を正確にたどります。
:::

## バスとサブグループ

**バス**は複数の信号を合算する共有の行き先です。最も重要なバスは**マスター** — すべてが到達する最終ステレオ出力です。ほかに 2 つの一般的なロールがあります。

- **AUX バス**はセンドのリターンです（共有リバーブはここに置きます）。
- **サブグループ**（サブミックス）は関連トラック — たとえばドラムすべて — をまとめ、1 つとして処理・調整できます。ドラムサブグループのコンプはキットをまとめ、1 本のフェーダーでキット全体のレベルを制御します。

**VCA グループ**は関連する考え方で、音声を再ルーティングせずに複数トラックのレベルだけを 1 本のフェーダーでまとめて調整します。パートを別々のバスへ流したまま調整したいときに便利です。

## オートメーション

静的なミックスが最良であることは稀です。**オートメーション**はコントロールを時間とともに変化させます。サビでボーカルを 0.5 dB 上げる、シンセを 4 小節かけてフェードアウトする、ライザーでフィルターを開く。コントロールは設定した点の間をカーブでたどります — 直線ランプ、なめらかな S カーブ、即時のジャンプ。これがミックスを静止させず呼吸させる方法です。

## メーター: 耳とともに目も信じる

耳は疲れ、部屋は嘘をつくので、ミキサーはメーターも見ます。最も重要なものは次のとおりです。

- **ピーク／トゥルーピーク** — 最高レベル。[トゥルーピーク](../true-peak.md)はサンプルメーターが見逃すサンプル間のオーバーを捉えます。
- **RMS／LUFS** — 平均レベルと知覚[ラウドネス](../lufs.md)。リスナーが実際に判断するのはこれです。
- **相関** — 左右チャンネルが一致しているか。強い負の値は、ミックスが[モノラル](./mono-compatibility.md)で弱まったり打ち消されたりする警告です。

:::: details libsonare がこれらをどうモデル化するか
libsonare のミキシングエンジンは、これらの概念を具体的なオブジェクトへ対応づけます。

| 概念 | 実装上の対応 |
|------|--------------|
| ストリップ | 入力トリム・EQ・フェーダー・パン・幅・インサート・センドを固定の信号順に持つ `ChannelStrip` |
| バス | `master`/`aux`/`submix` のロールを持つルーティング先 |
| センド | プリ／ポストフェーダーでタップする送信経路 |
| オートメーション | サンプル精度の位置で `linear`/`exponential`/`s-curve`/`hold` カーブとともにスケジュール |
| メーター | 各ストリップのメータースナップショットとゴニオメーター履歴バッファ |
| シーン | ミキサー全体を保存して読み直せる JSON データ |

エンジンはリアルタイム安全です。デノーマル対策、ロックフリーなパラメータ変更、事前確保バッファ、ルーティンググラフ全体のプラグインディレイ補償を備えるため、同じモデルがオフラインでも AudioWorklet 内でも動きます。
::::
